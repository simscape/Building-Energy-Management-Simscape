Building energy management is an important step when addressing energy goals and the transition to clean energy. Enhanced energy management in buildings supports the transition to renewable energy sources, aids in the global commitment to limit temperature rise, and combats climate change. In this project, you will create a system-level model using and to assess building HVAC needs and develop building energy management systems (BEMS). This project contains custom libraries, utilities, models, and code to help you analyze a building energy HVAC requirements and design controllers for BEMS. You will learn how to quickly create building models, add solar load to them, and create system models with HVAC components to simulate the building energy needs or controller design.

Modeling Approach

A typical workflow includes creating a building of your design and then analyzing it. This picture shows that process.

Create Building Model

See Create a building and add solar load to it, to learn how to create buildings with the help of utilities in the utils folder. You will also learn how to specify solar load to the building based on it's location and orientation. The building data is saved as a XML part which can be shared with other users or saved for future analysis. There are three different approaches to choose from. You can choose to create your own building by
  1. Using utilities provided in this project
  2. Using libraries provided in this project
  3. Importing your Building Information Model BIM into Simscape.

Setup Simscape Model

The building part data is saved and used to parameterize custom components in Simscape, build a Simscape based Building Energy Management model.
There are 2 Simscape models (SLX files in the Model folder) that can be parameterized.
  1. BuildingModelWithSolarLoad, used during HVAC requirement analysis
  2. BuildingModelWithSolarLoadAndHVAC, used during BEMS analysis
Both of these models use one or more of the 4 custom components provided in the project: Building, Ambient, Heat Pump, and Operational Data. Find these comoponents in Components/Model directory.The picture below shows the composition of these two models.
