Create Building Definition

Using MATLAB Utilities

addRoomToFloorPlan

This function adds room data to a floor plan.
Arguments:
  1. FloorPlan: Data struct for apartment unit floorplan.
  2. NewRoom: Name of the new room being added to the FloorPlan, specified as a string.
  3. Vertex: The point on the x-y plane that defines the room vertex or one of the corners, specified as a vector of Simscape values. It represents the coordinate locations in the x and y direction, or the east-west and north-south direction for the room starting point.
  4. Length: Length of the new room, specified as a Simscape value.
  5. Width: Width of the new room, specified as a Simscape value.
  6. Angle: Angle by which the room is rotated, specified as a Simscape value.
  7. Plot: An optional argument, specified as either true or false. The default value is false. When you specify true, the function plots the room being added along with other rooms already present in the FloorPlan data struct definition. The plot can help to visually confirm the room placement with reference to other rooms.
Example:
oneAptUnit = addRoomToFloorPlan(FloorPlan=oneAptUnit,...
NewRoom="LivingRoom",...
Vertex=simscape.Value([0,0],'ft'),...
Length=simscape.Value(8,'ft'),...
Width=simscape.Value(10,'ft'),...
Angle=simscape.Value(0,'deg'));

addOpeningOnWallSection

This function adds window and vent definition to an external wall in the building. The visualizeBuildingWallsToAddWindowsVents function represents the external wall edges and adds a number to them. These number are used to specify the wall portion to add a window or vent.
Arguments:
  1. FloorPlan: Building floorplan.
  2. Data: A table of 6 columns and number of rows equal to the number of openings you wish to specify. This table is auto generated as output of the function visualizeBuildingWallsToAddWindowsVents. Column 1 and 2 refer to indices of the wall section represented on the floorplan (see plot generated by visualizeBuildingWallsToAddWindowsVents). The 3rd column specifies the name of the room. The 4th and 5th column are specified as a scalar value between 0 and 1, which represents what fraction of total wall area that the opening occupies, for windows and vent respectively. The final column, specified as a string, contains the value of the apartment, room, and wall number.
Example:
updatedFloorPlan = addOpeningOnWallSection(FloorPlan=floorPlan,Data=tblOpening);

copyMoveRotateForNewApartment

This function can move, rotate, and create a copy of an existing apartment. Use this function to create copies of apartments that have similar layout.
Arguments:
  1. Apartment: Specify apartment data struct.
  2. MoveDistance: Specify distance vector for moving floorplan in XY plane.
  3. RotationAngle: Specify angle for rotation of floorplan.
  4. RotateAboutRoom: Name of the room around which the floorplan is to be rotate.
  5. Tol: Percentage value and used to detect if two sides of a room are overlapping or not. This value is used in auto-detecting common walls and helps in creating building thermal network in Simscape.
Example:
apartment = copyMoveRotateForNewApartment(Apartment=modelApt,...
MoveDistance=simscape.Value([0,0],'ft'),...
RotationAngle=simscape.Value(15,'deg'),...
RotateAboutRoom="Bedroom",...
Tol=0.01);

getApartmentRoomConnectivity

This function acquires the room-to-room connectivity data within an apartment. The output is a table that specifies the room-to-room connectivity. The function assumes that all rooms have solid walls between them. The table contains information on the connecting rooms and their Solid Wall Fraction values. A value of 1 denotes a solid wall between two adjacent rooms while a value of 0 denotes no wall between the two rooms. An intermediate value represents a door or an opening in the wall. To specify a gap or an opening, a door, window, or completely removed wall between the two rooms, edit the relevant Solid Wall Fraction field. To modify room-to-room connectivity in entire building, see getBuildingRoomConnectivity.
Arguments:
  1. Apartment: Data struct for apartment.
Example:
aptTbl = getApartmentRoomConnectivity(Apartment=apartment);

getBuildingRoomConnectivity

This function calculates the room-to-room connectivity in a building, and returns the connectivity as a table. The table contains information on the floor level, apartment number, room number, and the room name for all pairs of connected rooms. The final column is the Solid Wall Fraction parameter which specifies whether the common wall section between two rooms is a solid wall or has an opening or a gap. The value of Solid Wall Fraction varies between 0-1, with 1 representing a solid wall and 0 representing no wall. Any intermediate value represents doors or opening gap between the two rooms.
Arguments:
  1. Building: A data struct to specify building definition.
Example:
bldgTbl = getBuildingRoomConnectivity(Building=building);

plot3DlayoutBuilding

This function plots the 3D building and visualize it in colors specified by variables such as solar radiation, sunlight, and temperature. You can also plot just the walls, or just the roof elements too. You can also visualize the building in random colors, when solar radiation data has not been incorporated into building data struct or the data struct does not contain simulation results.
Arguments:
  1. Building: A struct containing 3D building data, generally the output from the function generateBuilding3Dlayout.
  2. PlotViewDirection: A vector, specifying the 3D plot viewing direction.
  3. ColorScheme: specify either simple, radiation, temperature, or random. If the building struct buildingData contains solar radiation data (obtained by use of function getHourlyBuildingSolarLoad), then you can use the radiation option to plot the building with incident solar radiation or incident sunlight at a particular time of the day, specfied by the final argument hourNum. If you have run simulations and updated building data struct with simulated temperature data, then you can use the temperature option to plot the temperature variation over the building with respect to time.
  4. Transparency: This argument is optional and is used to specify the transparency of the building plot. It takes any value between 0-1. The default value is 1, implying no transparency.
  5. Hour: Hours since the start of the simulation. This argument is optional when the ColorScheme is selected as random.
  6. InternalWalls: When set to true, the plot functions also displays the internal building walls. To display the internal walls, the Transparency value is set to 0.5, if not specified in the function using the name value pair Transparency. The default setting for this parameter is false.
  7. DebugMode: This mode switches on the text display for doors and windows/vents opening size visualization. The default setting for this parameter is false.
Example:
plot3DlayoutBuilding(Building=building,...
PlotViewDirection=viewingAngle,...
ColorScheme="simple",...
InternalWalls=false,...
DebugMode=false);

plotFloorPlanLayout

Use this function to plot a room, apartment, or floorplan.
Arguments:
  1. PlotData: Data struct for a room, apartment, or floorplan.
  2. Type: A string, which specifies content to be plotted. This string can be either "room", "apartment", or "floorplan. Specify the string based on the type of structure for dataStruct.
Example:
plotFloorPlanLayout(PlotData=floorPlan,Type="floorplan")

updateApartmentRoomConnectivity

This function updates the room-to-room connectivity data within an apartment. The input is the apartment data struct and the modified connectivity table data, obtained from getApartmentRoomConnectivity function. The function returns the updated data struct for the apartment.
Arguments:
  1. Apartment: Data struct for apartment.
  2. ConnectionTable: Table specifying the room-to-room connectivity data. This table format is an output of the function getApartmentRoomConnectivity.
Example:
apartment = updateApartmentRoomConnectivity(Apartment=apartment,...
ConnectionTable=aptTbl);

updateBuildingRoomConnectivity

This function uses the tabular output provided by getBuildingRoomConnectivity function to update the room-to-room connectivity data in a building. The table contains information on the floor level, apartment number, room number, and the room name for all pairs of connected rooms. The final column is the Solid Wall Fraction parameter that specifies whether the common wall section between two rooms is a solid wall or has an opening or a gap. Edit this filed value depending on your requirements and use the updated table to modify the building data struct.
Arguments:
  1. Building: A data struct to specify building definition.
  2. ConnectionTable: Table that specifies the room-to-room connectivity data for the entire building. This table is the output of the function getBuildingRoomConnectivity.
Example:
bldg = updateBuildingRoomConnectivity(Building=building,ConnectionTable=bldgTbl);

visualizeBuildingWallsToAddWindowsVents

This function visualizes the external walls of a building and adds a number for specifying the wall section. It also returns a table that specifies the external wall numbers along with the associated rooms. Default values of window and vent sizes are set to 0. You can change any value of window and vent to specify an opening on the external wall. The value provided must be a fraction (0-1); window size or the vent size is specified as the fraction multiplied with the total wall area. Also, see addOpeningOnWallSection for how to modify and add the table to the building description.
Arguments:
  1. FloorPlan: Building floorplan.
  2. Tol: Tolerance value.
  3. Plot: If set to true, outputs plot from the function.
Example:
tblOpening = visualizeBuildingWallsToAddWindowsVents(FloorPlan=floorPlan,Plot=true,Tol=0.01);

Import BIM Data

createBuildingDataFromBIM

This function helps create a Simscape building definition from the BIM geometry data extracted using the function getBuildingDataFromBIM. The output of this function is saved as the building part data.
Arguments:
  • DataBuilding: A struct containing all building geometry data. It is an output of the function getBuildingDataFromBIM.
  • DataBIM: A struct containing all building property data, as specified in the BIM model. It is an output of the function getBuildingDataFromBIM.
  • Debug: When set true, the function displays diagnostics from the function.
  • Tol: A scalar value to specify tolerance when checking for adjacent rooms or merging rooms for creating composite shape rooms.
  • BuildingName: A string, to specify the building name.
Example:
[building,modelData] = createBuildingModelFromBIM(...
DataBIM=dataBIM,...
DataBuilding=modelData,...
Debug=true,...
Tol=0.00001,...
BuildingName="MyBuilding");
% 'building' is the output struct that is saved as Building part definition.

getBuildingDataFromBIM

This function helps extract building data from a BIM model exported to a XML format. This function used Revit data exported onto a XML. The data is processed and extracted in a form that the Simscape utilites can understand to create a building model.
Arguments:
  • FileName: Name of the XML file for BIM data.
  • PhysUnitMappingFile: The file containing unit mapping data from Revit to Simscape. This is optional argument and it links by default to a *.xlsx file saved in the project.
  • PhysUnitMappingRowSize: The maximum number of entries (rows) in the PhysUnitMappingFile data.
  • Debug: When set to true, it outputs key diagnostics data from the function execution.
  • RoundOff: Round off value for computation done is the utility functions.
Example:
[modelData,dataBIM] = getBuildingDataFromBIM(FileName="importBuildingTshapedRoom.xml",...
Debug=false,RoundOff=5);

Create Building Library

createBuildingLibraryAndParameterize

This function creates a Simscape building library and provides you an option to save it in your project. This library can be used in different projects for engineering analysis.
Arguments:
  • BuildingPartFile: The XML part file with building data.
  • BuildingLibraryName: A string to specify name for the library.
  • ModelType: Library model variant, Select Heat Load Analysis to build a Thermal components based library (thermal, no flow). Select Building Energy System (TL) to build a Thermal Liquid components based library (thermal and flow).
  • Diagnostics: When set to true, outputs key messages or progress of the build process.
  • SaveModel: Specify options for the library created. Select Create Model to view the created library. This does not save the library in your project. You must do that manually, if you are satisfied with the output. This is useful when you are iterating with library choices and want to quickly check out a few options. Select Create Model and Save to create the library and save the model in ScriptsData/BuildingLib location. Select Create Model and Save With Icon to create the library, it's icon, and save it at ScriptsData/BuildingLib location. Select Create Model, Save With Icon, and Update Path to create the library, it's icon, and save it at ScriptsData/BuildingLib location. This option also adds the library to the project path. With other options, you must add the library to the project path, manually.
  • WallMaterialTable: Table of wall material data, generated from initializeBuildingData function.
  • GroundThermalRes: A simscape value to specify ground thermal resistance.
  • IniBuildingTemperature: A simscape value to specify building initial temperature.
  • HeatTransferCoeffInt: A simscape value to specify heat transfer coefficient within the building rooms.
  • HeatTransferCoeffExt: A simscape value to specify heat transfer coefficient from ambient to the building walls.
  • PipingDataTable: Table of piping data, generated from initializeBuildingData function. This is optional and needed when building library of Thermal Liquid variant is being built.
Example:
modelData.buildingPartFile = "BuildingGeometryDataFile.xml";
modelData.myLibraryName = "BuildingHeatLoadAnalysis";
 
bldgBlockPath = createBuildingLibraryAndParameterize(...
ModelType="Heat Load Analysis",...
BuildingPartFile=modelData.buildingPartFile,...
BuildingLibraryName=modelData.myLibraryName,...
IniBuildingTemperature=simscape.Value(300,"K"),...
GroundThermalRes=simscape.Value(1,"K/W"),...
HeatTransferCoeffExt=simscape.Value(20,"W/(K*m^2)"),...
HeatTransferCoeffInt=simscape.Value(30,"W/(K*m^2)"),...
WallMaterialTable=wallMatProp,...
SaveModel="Create Model",...
Diagnostics=true);

getGeoLocationForMajorCities

This function defines the longitude and the latitude for well known cities in the world. This data is required to compute the solar radiation value. If your city is not available in the list, you can create an empty template and add data to it.
Arguments:
  • CityName: A string, denoting the name of the city.
Example:
geoLocation = getGeoLocationForMajorCities(CityName="Helsinki");
 
% If your city name is not present in the list, then
% you may enter it manually, as shown below:
 
%% Create a blank template
geoLocation = getGeoLocationForMajorCities();
 
%% Add Data to The Blank Tempate
% Add name of the city
geoLocation.Row = "MyCityName";
% Add longitude value
geoLocation.Longitude = {'5W'};
% Add latitude value
geoLocation.Latitude = {'45N'};
% Add Meridian value for local time zone calculation
geoLocation.("Meredian (Time Zone)") = {'5W'};
% Check if the values look ok.
disp(geoLocation);

initializeBuildingData

This function initializes wall material property table and piping data table for the building library component.
Arguments:
  • ModelType: Library model variant, Select Heat Load Analysis to build a Thermal components based library (thermal, no flow). Select Building Energy System (TL) to build a Thermal Liquid components based library (thermal and flow).
All other parameters are optional. Some default values are set to them and can be modified using table operations.
  • RoofMaterial, InternalWallMaterial, FloorMaterial, ExternalWallMaterial: Select material for walls from one of the following - Wood,Brick,Concrete,ClinkerBrick. You can make changes to table values if you do not want to use material from these four selections.
  • RoofThickness, InternalWallThickness, FloorThickness, ExternalWallThickness: Specify simscape value for these wall properties.
  • WindowGlassThickness: A simscape value to specify window glass thickness.
  • DiameterDistributor, DiameterRadiator, DiameterUnderFloor: Simscape values for the pipe radius used in fluid distribution pipe, radiator piping, and underfloor piping for HVAC. These values are used to populate the piping data table.
  • GapRadiatorPipe, GapUnderFloorPipe: A simscape value to specify the gape between adjacent pipe centerlines in a radiator and underfloor pipe network. These values are used to populate the piping data table.
  • Diagnostics: If set to true, outputs key tasks accomplished during the function call.
Example:
[wallMatProp,pipingData] = initializeBuildingData(...
ModelType="Heat Load Analysis",...
ExternalWallMaterial="Concrete",...
ExternalWallThickness=simscape.Value(0.07,"m"),...
InternalWallMaterial="ClinkerBrick",...
InternalWallThickness=simscape.Value(0.04,"m"),...
FloorMaterial="Brick",...
FloorThickness=simscape.Value(0.03,"m"),...
RoofMaterial="Concrete",...
RoofThickness=simscape.Value(0.04,"m"),...
WindowGlassThickness=simscape.Value(0.005,"m"),...
Diagnostics=true);

Create Analysis Template

setUpBuildingModel

This function creates a model template for analysis, based on the buildig custom library block selected.
Arguments:
  • BuildingLibName: A string to specify the library name.
  • Diagnostics: When set to true, outputs key updates from the model build process.
  • LibDisplaySize: An optional argument, specified as an integer to define the library block size. Default value is set to 100.
  • LibStartPositionOnCanvas: A column vector for the (x,y) location for library start point. Default value is set to [400,200].
  • AddEnvironmentBlocks: An optional argument, which when set to true, sets up environment related custom blocks on the model canvas.
  • EnvLibDisplaySize: An optional argument, specified as an integer to define the environment related block size. Default value is set to 50.
  • StartTime: A datetime parameter to specify the start time.
  • EndTime: A datetime parameter to specify the end time.
  • DayLightSavingHours: Number of hours of daylight savings.
  • GeoLocation: A table data to specify the geographical location of the building. This table is obtained from getGeoLocationForMajorCities.
  • DayTimeTemperature: A simscape value for average daytime temperature. Specified as a vector of length equal to the number of days set for simulation (using name-value-pair StartTime and EndTime). If the value is set to a scalar quantity, then all days are assumed to be at the same temperature.
  • DayTimeTemperatureVar: Specified as a scalar value representing the percent deviation in day-time temperature from the value set in name-value-pair DayTimeTemperature.
  • NightTimeTemperature: A simscape value for average nighttime temperature. Specified as a vector of length equal to the number of days set for simulation (using name-value-pair StartTime and EndTime). If the value is set to a scalar quantity, then all nights are assumed to be at the same temperature.
  • NightTimeTemperatureVar: Specified as a scalar value representing the percent deviation in night-time temperature from the value set in name-value-pair NightTimeTemperature.
Example:
modelData.location = getGeoLocationForMajorCities(CityName="Vienna");
modelData.startDate = datetime(2024,1,1,1,0,0);
modelData.endDate = datetime(2024,1,20,1,0,0);
modelData.nDays = days(modelData.endDate-modelData.startDate);
modelData.nHour = hours(modelData.endDate-modelData.startDate);
 
modelData.dayT = simscape.Value(275*ones(1,modelData.nDays),"K");
modelData.nightT = simscape.Value(271*ones(1,modelData.nDays),"K");
 
setUpBuildingModel(NewModelName=modelData.mySystemModel,...
BuildingLibName="BuildingHeatLoadAnalysis",...
Diagnostics=true,...
GeoLocation=modelData.location,...
StartTime=modelData.startDate,...
EndTime=modelData.endDate,...
DayTimeTemperature=modelData.dayT,...
NightTimeTemperature=modelData.nightT);

Datacenter Model

buildDatacenter

This function is used to connect multiple datacenter servers in a certain topology. There a 3 different variants available, thermal, electrical, and electrothermal.
Parameters:
  • NumPDUunitsX: Number of power distribution units (PDU) of datacenter in X direction.
  • NumPDUunitsY: Number of power distribution units (PDU) of datacenter in Y direction.
  • ModelName: Name of Simscape model.
  • ModelOption: Model fidelity; 3 different options available - thermal, electrical, and electrothermal.
  • RatingDatacenter: A simscape value to specify desired power rating of the datacenter.
  • PowerSupplyEff: A scalar to specify the percentage efficiency of power supply. This is an optional argument required for Electrical and Electrothermal model fidelities.
  • ThermalRes: A scalar to specify the thermal resistance of the datacenter stack bottom rack and the ground.
  • Diagnostics: A logical, when set to true, outputs key updates from the function/utility.
Example:
datacenter = buildDatacenter(NumPDUunitsX=5,...
NumPDUunitsY=6,...
ModelName="MyModel",...
ModelOption="Electrothermal",...
RatingDatacenter=simscape.Value(50,"kW"),...
PowerSupplyEff=85,...
ThermalRes=simscape.Value(5,"W/K"),...
Diagnostics=true);

buildEnclosureForDatacenter

This function helps build a rectangular enclosure or room around the datacenter.
Arguments:
  • ModelName: Specified as a string and denotes the name for the Simscape model.
  • EnclosureName: Specified as a string and denotes the name for the room/enclosure subsystem.
  • DatacenterModel: A struct specifying datacenter properties.
  • Diagnostics: When set to true, outputs key updates from the build process.
  • RoomLength, RoomWidth, RoomHeight: Simscape values for room/enclosure length, width, and height.
  • WallThickness: Simscape value to specify the wall thickness for the room/enclosure.
  • RoomRotate: Simscape value for the value of which the room is to be rotated.
  • PercentAirVolInRoom: Percent volume of room that is empty or has stagnant air.
Example:
roomModel = buildEnclosureForDatacenter(ModelName=NameValueArgs.ModelName,...
DatacenterModel=datacenter,...
EnclosureName=NameValueArgs.EnclosureName,...
PercentAirVolInRoom=8,...
RoomLength=simscape.Value(4,"m"),...
RoomWidth=simscape.Value(3.5,"m"),...
RoomHeight=simscape.Value(2.5,"m"),...
WallThickness=simscape.Value(1e-1,"m"),...
RoomRotate=simscape.Value(15,"deg"),...
Diagnostics=false);

Engineering Analysis

getMonthlyAveragedData

This function is used to find average monthly values of certain quantities over long period of time, typically over few months. It returns the values in form of a plot as well as a tabular data.
Arguments:
  1. DateTimeVec: A datetime vector of period of interest.
  2. Data: A row vector of length equal to the length of the DateTimeVec parameter.
  3. Plot: If set to yes, it plots the monthly averaged value of the quantity specified in vector Data. The defult value is set to false.
  4. PlotName: A string, if specified, displays the text in plot title and on the y-label text. The defult is set to "data".
  5. DisplayData: If set to yes, it displays the monthly averaged value of the quantity specified in vector Data in a table. The defult value is set to false.
Example:
getMonthlyAveragedData(DateTimeVec=dateTimeVec,Data=Q,Plot=true,PlotName="Heating Requirements (W)",DisplayData=true);

getTemperatureFromYearlyData

This function constructs daily temperature data from the monthly average data of temperature for a year. The function requires minimum and maximum temperature values for each month, specified as a vector and it outputs the daily average temperature.
Arguments:
  • DateTimeVec: Datetime vector or timestamps.
  • Location: Location data, specified as longitude and latitude data in a table.
  • Tavg12MonthData: A 12x2 matrix of Simscape values containing 12-month temperature data and the minimum and maximum temperature during a month. Column 1 represents the minimum temperature and column 2 represents the maximum temperature during the month.
  • DayLightSavingHrs: Day light saving hours specified as a scalar value.
  • DeltaDegreesNightT: Approximate difference between day and night time temperature, specified as a scalar Simscape value.
  • PercentDayTimeTvar: Percentage variation in day temperature in a single day, specified as a scalar.
  • PercentNightTimeTvar: Percentage variation in night temperature in a single day, specified as a scalar.
  • Plot: Option to plot the temperature data, specified as boolean.
  • PlotName: Optional string that specifies the name and the title of the plot.
Example:
[avgDayT,avgDayTvar,avgNightT,avgNightTvar] = ...
getTemperatureFromYearlyData(DateTimeVec=dateTimeVec,...
Tavg12MonthData=monthlyTavgMat,DeltaDegreesNightT=simscape.Value(3,"K"),...
Plot=true,PlotName="Helsinki",Location=geoLocation,...
PercentDayTimeTvar=1,PercentNightTimeTvar=1);

initializeBuildingMaterialProperties

This function initializes the values for building material properties. The function returns a table buildingMaterial that contains roof, floor, external walls, internal walls, and window properties. The properties include material thickness, density, heat capacity, thermal conductivity, and absorptivity (where applicable) and transmissivity (where applicable).
Arguments:
  1. DisplayData: A value, set as true or false, that specifies whether to display the final table. The default value for this is true.
Example:
buildingMaterial = initializeBuildingMaterialProperties(DisplayData=true);
